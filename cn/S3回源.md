# AWS S3 回源

> 入口：CDNPro 控制台 → **Properties** → 新建或编辑 Property

把 AWS S3 桶配置为 CDNPro 源站。S3 不是单独的"源站类型"，而是普通 HTTP/HTTPS 源站 + 在高级配置中开启 **认证 = AWS S3**，由边缘节点完成 SigV4 签名。

---

## 1. 物料准备

| 物料 | 说明 |
| --- | --- |
| S3 桶名 | 例：`my-bucket` |
| 桶所在 Region | 例：`us-east-1` |
| 桶访问域名 | `my-bucket.s3.us-east-1.amazonaws.com` |
| Access Key ID | IAM 的 AK |
| Secret Access Key | 对应 SK，录入到保密信息（Vault） |
| 加速域名 | 已完成证书上传 |
| 调度域名 | `xxx.qtlcdn.com`，由 **Edge Hostnames** 创建 |

---

## 2. 步骤

### 2.1 创建调度域名（Edge Hostname）

**Edge Hostnames** → **Create** → 选择 TLS 证书 → 生成 `xxx.qtlcdn.com`。

### 2.2 录入保密信息（Vault）

**Configuration** → **Secrets / Vault** → **Add Secret**：

- **Name**：自定义，如 `s3-my-bucket-sk`
- **Type**：`Secret String`
- **Value**：Secret Access Key

### 2.3 创建 Property

1. **Hostname**：业务加速域名，并选择调度域名与已上传证书。
2. **Origin**：
   - **Origin Type**：`HTTPS`
   - **Origin Hostname**：`my-bucket.s3.us-east-1.amazonaws.com`
   - **Origin Port**：`443`
3. **Host Header**：设为桶访问域名 `my-bucket.s3.us-east-1.amazonaws.com`。否则 S3 返回 `Bucket not found`。
4. **Advanced Configuration → Origin Authentication**：
   - **Authentication Type**：`AWS S3`
   - **Region**：桶所在 Region
   - **Access Key ID**：AK
   - **Secret Access Key**：选择 §2.2 录入的 Vault 条目

### 2.4 发布

提交后 **Deploy to Staging** → 验证 → **Deploy to Production**。

业务 DNS 把加速域名 CNAME 到 `xxx.qtlcdn.com`。
