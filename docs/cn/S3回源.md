# AWS S3 存储桶加速

> 来源：Wangsu Documentation → CDN Pro → Recipes → AWS S3 Bucket Acceleration（最后更新 2026-05-28）

CDN Pro 支持与 AWS S3 配合使用，实现资源云端加速分发与存储；特别地，CDN Pro 也支持访问私有 AWS 存储桶。本文以一个完整示例介绍如何在 AWS 与 CDN Pro 上配置 AWS S3 存储桶加速。

**示例场景**：我们有一个域名 `waytoo.digital`，希望在子域名 `files.waytoo.digital` 上挂一个 AWS S3 存储桶，让用户能通过该子域名访问桶里的文件资源。下面先演示访问公开桶，再演示访问私有桶。

## 步骤 1：创建并配置公开的 AWS S3 存储桶

1. 登录 AWS S3 控制台并创建一个新存储桶。

   ![创建存储桶向导](../../assets/s3/s3-01.png)

2. 输入存储桶名称（合法的 DNS 前缀）：`files-waytoo-digital`。

3. 单击下一步 **Next**。

4. 跳过配置选项 **Configure options**。

5. 默认勾选阻止所有公开访问的权限 **Block all public access**。
   > 创建存储桶时 AWS 默认希望你创建私有桶，可在创建完成后再设置为公开桶；否则 AWS 不允许进入下一步 **Next**。

6. 单击 **Next** → **Save**，存储桶创建成功。

7. 为新建的存储桶配置内容访问控制权限。在权限配置面板 **Permissions** 中：

   ![权限面板](../../assets/s3/s3-02.png)

8. 取消勾选 **Block all public access**，然后单击 **Save**。

   ![取消阻止公开访问](../../assets/s3/s3-03.png)

9. 复制以下 CORS 配置，粘贴到 **CORS configuration** 编辑器中：

   ```xml
   <?xml version="1.0" encoding="UTF-8"?>
   <CORSConfiguration xmlns="http://s3.amazonaws.com/doc/2006-03-01/">
   <CORSRule>
       <AllowedOrigin>*</AllowedOrigin>
       <AllowedMethod>GET</AllowedMethod>
       <MaxAgeSeconds>3000</MaxAgeSeconds>
       <AllowedHeader>Authorization</AllowedHeader>
   </CORSRule>
   </CORSConfiguration>
   ```

10. 上传文件到存储桶以测试桶配置，并对上传对象授予读写访问权限。

    ![上传文件](../../assets/s3/s3-04.png)

    ![上传授权](../../assets/s3/s3-05.png)

11. 点击桶中上传的文件，查看上传对象的 URL（**Object URL**）。

    ![Object URL](../../assets/s3/s3-06.png)

    Amazon S3 默认按一定格式生成访问桶文件的 URL。本例中得到的文件 URL 是
    `https://files-waytoo-digital.s3.amazonaws.com/file_1m.bin`。
    Amazon S3 文件的访问域名格式一般为 `<BUCKET_NAME>.<REGION>.amazonaws.com`；本例中 Amazon S3 没有返回区域字段，而是直接返回 `s3`。无论如何，我们可以使用这个桶域名作为 CDN Pro 的回源访问地址，稍后将用到 `files-waytoo-digital.s3.amazonaws.com`。

## 步骤 2：创建并配置测试 CDN Pro 加速项目

登录 CDN Pro 控制台，按"快速入门"逐步配置并验证测试加速项目。

### 2.1 配置调度域名

1. 在调度域名页面创建一个边缘调度域名 `files-waytoo-digital.qtlcdn.com`。

   ![创建调度域名](../../assets/s3/s3-07.png)

2. 创建后在终端验证它是否可解析：

   ```
   $ ping files-waytoo-digital.qtlcdn.com
   PING files-waytoo-digital.qtlcdn.com (14.0.119.186): 56 data bytes
   64 bytes from 14.0.119.186: icmp_seq=0 ttl=56 time=3.775 ms
   64 bytes from 14.0.119.186: icmp_seq=1 ttl=56 time=3.953 ms
   64 bytes from 14.0.119.186: icmp_seq=2 ttl=56 time=3.739 ms
   64 bytes from 14.0.119.186: icmp_seq=3 ttl=56 time=4.733 ms
   ```

### 2.2 配置证书

1. **创建证书**。在证书页面点击 **新建证书**，使用 Let's Encrypt 自动更新的方式创建一本新证书：
   1. 填写 **证书名称** 与 **证书说明**；
   2. 将 **自动更新** 设置为 **Let's Encrypt**，**创建方法** 选 **自动生成**，**公钥算法** 选 **RSA2048**；在 **授权域名** 与 **SAN** 字段中分别输入 `files.waytoo.digital`；
   3. 点击 **保存** 创建证书。

   ![创建证书](../../assets/s3/s3-08.png)

2. **将证书部署到演练环境**。证书创建后，先部署到演练环境进行测试，确认无误再发布到生产。

   ![部署证书](../../assets/s3/s3-09.png)

### 2.3 配置加速项目

1. **创建加速项目**。在加速项目页面点击 **新建加速项目**：
   1. 输入 **项目名称**、**项目描述**、**服务类型**；
   2. 在 **加速域名** 中输入至少一个域名。CDN Pro 会从客户端请求的 Host 头中提取该域名以匹配加速项目配置。

   ![新建加速项目](../../assets/s3/s3-10.png)

2. **添加 Amazon S3 存储桶源站**。在 **源站** 下点击 **添加**：
   1. **源站名称**：设为 `aws_origin`，稍后会在边缘逻辑配置中引用；
   2. **主机名/IP 地址**：填 `files-waytoo-digital.s3.amazonaws.com`，并点击 **验证** 检查源站联通性；
   3. **Host 请求头**：展开 **高级配置**，在 **Host 请求头** 字段填 `files-waytoo-digital.s3.amazonaws.com`；
   4. 点击 **保存**，完成 Amazon S3 源站创建。

   ![添加 S3 源站](../../assets/s3/s3-11.png)

   > **注意**：HOST 请求头必须配置为 Amazon S3 Bucket 的文件访问域名。如果缺省该配置，CDN Pro 回源到 Amazon S3 时 Host 头会默认携带加速域名 `files.waytoo.digital`，此时 Amazon S3 会响应 **Bucket not found**。

3. **边缘逻辑配置**。在边缘逻辑配置页点击 **默认缓存模板**，选择导入"全站加速"基础配置：
   1. 缓存模板中所有路径资源的源站均选 `aws_origin`；
   2. 指定路径缓存中勾选"强制忽略来自源站的请求头"，以控制 CDN Pro 上的缓存行为；
   3. 点击 **确定** 完成模板导入。可在编辑器中看到生成的边缘逻辑脚本（也可自行修改以匹配业务）。

   ![边缘逻辑配置](../../assets/s3/s3-12.png)

   ```nginx
   location / {
       proxy_cache_bypass 1;
       proxy_no_cache 1;
       origin_pass aws_origin;
   }
   location ~* \.(exe|wmv|mp3|wma|ogg|swf|flv|mp4|avi|mpg|mpeg|f4v|rmvb|rm|3gp|img|cur|gif|png|jpe|ico|bmp|jpeg|jpg|rpm|deb|tar|gz|bin|bz2|zip|rar|ipa|apk|jar|sis|xap|msi|cab|7z|pdf|doc|docx|xls|xlsx|ppt|pptx|txt|css)$ {
       set $cache_misc $cache_misc$sorted_querystring_args;
       if ( $args = '' )
       { set $cache_time 24h; }
       else
       { set $cache_time 10m; }
       proxy_cache_valid 200 206 $cache_time;
       proxy_ignore_headers Expires Cache-Control Set-Cookie;
       origin_pass aws_origin;
   }
   location ~* \.(html/htm/js)$ {
       set $cache_misc $cache_misc$sorted_querystring_args;
       if ( $args = '' )
       { set $cache_time 12h; }
       else
       { set $cache_time 0m; }
       proxy_cache_valid 200 206 $cache_time;
       proxy_ignore_headers Expires Cache-Control Set-Cookie;
       origin_pass aws_origin;
   }
   location ~* \.(shtml/html/htm/js)$ {
       set $cache_misc $cache_misc$sorted_querystring_args;
       if ( $args = '' )
       { set $cache_time 12h; }
       else
       { set $cache_time 0m; }
       proxy_cache_valid 200 206 $cache_time;
       proxy_ignore_headers Expires Cache-Control Set-Cookie;
       origin_pass aws_origin;
   }
   ```

4. **为加速项目配置证书**。为加速项目配置一本 TLS 证书以支持 HTTPS 安全分发。在 TLS 配置页点击 **TLS 证书** 字段，下拉选择刚刚创建并部署在演练环境中的证书。

   ![TLS 证书绑定](../../assets/s3/s3-13.png)

5. **保存 / 验证 / 测试**。点击页面下方 **保存** 提交配置。如保存与验证均无问题，可将加速项目部署到演练环境进行测试。

   ![保存与部署](../../assets/s3/s3-14.png)

### 2.4 在演练环境中测试

要在演练环境中测试，先获取演练环境服务器的 IP：

```
$ dig staging.qtlgslb.com

```

```
content-length: 1024000

x-amz-request-id: 4Z0SEN8YER8PBYAG
date: Fri, 16 Oct 2020 03:30:16 GMT
last-modified: Fri, 16 Oct 2020 02:36:45 GMT
etag: "fcaebe20d9b3e73086dcb0ba5def8e52"
x-cache-status: MISS

```

再次请求相同对象，由于 CDN Pro 已有缓存，缓存状态为命中 **HIT**：

```
❯ curl -I https://files.waytoo.digital/file_1m.bin --resolve files.waytoo.digital:443:163.171.228.89
HTTP/2 200
...
x-cache-status: HIT
...
```

测试配置如期工作后，即可部署到生产环境。

### 2.5 部署到生产

要将配置部署到生产环境，需先到 **证书页面** 与 **加速项目页面** 下，分别将其部署到生产。

![部署到生产](../../assets/s3/s3-15.png)

### 2.6 在生产环境中测试

在切量 DNS 之前，可先在生产环境验证配置是否如期运行；响应正常即说明部署成功：

```
❯ curl -I https://files.waytoo.digital/file_1m.bin --resolve files.waytoo.digital:443:14.0.119.186
HTTP/2 200
content-type: application/octet-stream
content-length: 1024000
...
x-cache-status: HIT
x-via: 2.0 as-kr-icn1-cache-0003 [HIT]
...
```

业务测试通过后，创建/修改域名的 DNS CNAME 记录，指向之前创建的边缘调度域名 `files-waytoo-digital.qtlcdn.com`：

```
❯ dig files.waytoo.digital
...
;; ANSWER SECTION:
files.waytoo.digital.            299 IN CNAME  files-waytoo-digital.qtlcdn.com.
files-waytoo-digital.qtlcdn.com.  19 IN A      14.0.119.186
...
```

至此，所有访问 `files.waytoo.digital` 的请求都会经 CDN Pro 加速分发，回源到 Amazon S3 Bucket `files-waytoo-digital.s3.amazonaws.com` 获取文件资源。

## 步骤 3：访问私有存储桶

前面已创建一个公开桶并实现加速分发。现在把这个桶设为私有桶，并配置 CDN Pro 以授权方式访问它。

### 3.1 将 Amazon S3 Bucket 设置为私有存储桶

回到 AWS S3 的存储桶配置页面，在权限设置 **Permissions** 下勾选 **Block all public access**，将存储桶设为私有桶。此后从私有桶获取文件资源时必须携带密钥，AWS S3 鉴权通过后才允许访问。

![设置为私有桶](../../assets/s3/s3-16.png)

### 3.2 CDN Pro 上开启回源身份验证

编辑加速项目，设置 AWS S3 回源鉴权。在此之前需要先从 AWS S3 获取以下信息：

- **AWS 存储桶地区**
- **AWS 访问密钥 ID（Access Key）**
- **AWS 访问密钥加密串（Secret Key）**

1. 在加速项目页面选择此前创建的加速项目，**克隆出一个新版本**。
2. 编辑这个版本，并修改前面设置的 `aws_origin` 源站。
3. 在源站 **高级配置** 中，开启 AWS S3 回源鉴权：
   1. **认证**：选 **AWS S3**；
   2. **地区**：选 AWS S3 Bucket 的存储地区；
   3. **Access Key**：填写 AWS S3 访问密钥 ID；
   4. **Secret Key**：选择存放访问密钥加密串的 **保密信息**。

   填写完成后点击 **保存**。

   ![AWS S3 回源鉴权](../../assets/s3/s3-17.png)

4. 将加速项目版本部署到演练环境，验证运行符合预期：

   ```
   curl -I https://files.waytoo.digital/file_1m.bin --resolve files.waytoo.digital:443:163.171.228.89
   HTTP/2 200
   ...
   x-cache-status: HIT
   ...
   ```

5. 部署到生产环境。

到此完成 AWS S3 私有桶配置，并实现其资源的加速分发。
